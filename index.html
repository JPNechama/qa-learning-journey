My Weekly Learning Plan

press here to my course
  • Week 0: HTML Basics

    Learn the structure of web pages using HTML. Create your first simple site with headings, lists, links, and forms. This will help you later when testing web applications.

  • Week 1: Introduction to QA

    Understand what software is, what QA means, and explore the development process in hi-tech companies.

  • Week 2: Types of Testing

    Learn about different testing levels and methods, and practice identifying various testing types.

  • Week 3: Writing Test Documentation

    Start writing professional test documents such as STD (test design), STR (test report), and STP (test plan).

  • Week 4: Bug Reporting

    Practice writing high-quality bug reports and managing defects in tools like Jira. Learn the Bug Life Cycle.

  • Week 5: Web Testing

    Explore web technologies, write test trees, and practice testing websites while reporting bugs.

  • Week 6: Mobile Testing

    Learn the differences between testing mobile apps and web apps, and practice testing mobile projects.

  • Week 7: Test Management

    Use TestRail and Jira to manage test cases and organize test runs like a professional QA team.

  • Week 8: Databases and SQL

    Learn how to use SQL queries to validate data and support your testing process.

  • Week 9: API Testing

    Get started with server-side testing using XML, JSON, and tools for validating API responses.

  • Week 10: Preparing for Your First Job

    Build your CV, prepare for interviews, and learn how to market yourself as a QA professional.

This plan is my roadmap: step by step, week by week, I will grow from a beginner into a confident QA Automation professional, ready to take on real projects and start my new career.
